### 业务领域约束

#### 约束等级
- 🚫 强制 — 类型与格式规范
- ⚠️ 警告 — 外部调用与状态管理
- ✅ 验证 — 完成后检查

#### 使用范围

当变更涉及金额、时间、状态流转、外部依赖调用、并发编排时应用本规则。

#### 1. 通用领域规则

**🚫 强制规范：**
| 领域 | 规范 | 错误示例 | 正确示例 |
|------|------|----------|----------|
| 金额 | 使用 `int64`，单位为分 | `float64 amount` | `int64 amount` (分) |
| 时间 | 使用 `time.Time` | `int64 created_at` | `time.Time CreatedAt` |
| 状态 | 通过集中校验的状态迁移变更 | `user.Status = 2` | `sm.Transition(u, EventApprove)` |

**⚠️ 警告触发：**
- 外部接口调用 → 必须设置超时（默认 3s，使用 `context.WithTimeout`）并定义失败后的处理策略
- 状态变更 → 必须通过集中状态迁移校验，禁止直接修改状态字段

#### 2. 判定边界

**金额：**
- 金额、余额、手续费、优惠、授信额度、应收应付等字段统一使用 `int64`
- 单位必须在字段注释、结构体注释或 spec 中明确为“分”
- 仅在展示层做格式化时允许转换为字符串或小数形式，禁止将 `float64` 作为持久化或业务计算主类型

**时间：**
- 业务对象字段、函数入参与返回值默认使用 `time.Time`
- 只有在协议层、存储层适配第三方格式时，才允许临时使用时间戳；进入领域逻辑前必须转换

**状态变更：**
- “通过状态机变更”不强制要求引入专门库，但必须存在一个集中入口或校验函数表达合法迁移
- 下面这些不算合规：
  - 直接 `obj.Status = next`
  - 在多个文件分散写 `if old == x { status = y }`
  - 仅定义枚举常量但没有迁移校验

**外部调用：**
- 包括 HTTP、gRPC、RPC、消息发送、数据库之外的第三方 SDK、跨进程调用
- 最低要求：
  - 透传 `context.Context`
  - 设置明确超时；未有项目约定时默认 3 秒
  - 明确失败处理策略：重试、降级、返回错误、缓存兜底、跳过后异步补偿，至少说明一种
- “做降级处理”不等于必须吞错；直接向上返回错误也可以，只要 spec 或代码中说明这是预期策略

**✅ 验证清单：**
- [ ] 金额字段是否为 `int64` 类型且单位为分
- [ ] 时间字段是否为 `time.Time` 类型
- [ ] 外部调用是否设置了超时，并说明失败处理策略
- [ ] 状态变更是否通过集中校验入口而非直接赋值

#### 3. 推荐示例

**金额字段：**
```go
type Order struct {
    TotalAmount int64 // 单位：分
}
```

**外部调用超时：**
```go
ctx, cancel := context.WithTimeout(ctx, 3*time.Second)
defer cancel()

resp, err := client.Do(ctx, req)
if err != nil {
    return nil, fmt.Errorf("call payment service: %w", err)
}
```
