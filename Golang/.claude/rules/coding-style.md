### 编码规范

#### 约束等级
- 🚫 强制 — 必须遵守，违反会引入缺陷
- ⚠️ 警告 — 应避免，特殊情况需注明原因
- ✅ 验证 — 完成后自检

#### 1. 命名规范

**🚫 强制规则：**
| 对象 | 规范 | 禁止 |
|------|------|------|
| 文件名 | 全小写，下划线分隔 | `userService.go`, `UserService.go` |
| 结构体/接口 | 大驼峰，接口以 `er` 结尾 | 拼音、中英混拼 |
| 变量/函数 | 驼峰，导出用大写 | 拼音、中英混拼 |
| 常量 | 大驼峰或全大写下划线 | 拼音 |
| 测试文件 | `*_test.go`，函数以 `Test` 开头 | 其他命名 |

#### 2. 错误与异常处理

**🚫 强制规则：**
- 业务错误 → 必须使用自定义 Error 类型，通过返回值传递
- 禁止滥用 `panic`
- 错误透传 → 必须使用 `fmt.Errorf("... %w", err)` 包装
- 禁止吞掉错误（禁止 `_ = err` 或只打日志不 return）

**⚠️ 警告触发：**
- 独立 Goroutine 入口 → 必须使用 `defer recover()` 兜底

**✅ 验证清单：**
- [ ] 错误是否通过返回值传递而非 panic
- [ ] 错误是否使用 `%w` 包装保留堆栈
- [ ] 是否有 `_ = err` 或日志后无 return
- [ ] Goroutine 是否有 recover 兜底

#### 3. 日志规范

**🚫 强制规则：**
- 禁止在日志中打印用户敏感信息（手机号、身份证、银行卡等）

**✅ 验证清单：**
- [ ] Handler/Controller 入口是否打 INFO（含请求关键参数）
- [ ] 错误是否打 ERROR（含 traceID）
- [ ] 是否打印敏感信息

#### 4. 其他规范

**🚫 强制规则：**
- 魔法值必须定义为常量
- 写接口必须考虑幂等

**⚠️ 警告触发：**
- 并发场景 → 必须说明同步策略（`sync.Mutex` 或 `channel`）

**✅ 验证清单：**
- [ ] 是否有魔法值未定义为常量
- [ ] 接口是否考虑了幂等
- [ ] 并发场景是否说明了同步策略
