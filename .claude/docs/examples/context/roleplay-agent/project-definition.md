# 项目定义

## 1. 项目概述

- 项目名称：Roleplay Agent
- 一句话目标：为用户提供可配置角色与场景的对话练习平台
- 为什么要做：用户需要在低成本、可重复的环境里练习特定沟通场景
- 当前要解决的问题：现有练习方式缺少稳定角色、一致场景和结构化反馈
- 什么算成功：用户能完成一轮角色练习并获得清晰反馈

## 2. 目标用户与场景

- 目标用户：需要进行话术训练、面试练习或情境沟通演练的个人用户
- 谁操作 / 谁受益：练习者本人操作并受益
- 典型场景：用户选择角色模板，输入练习目标，进入多轮对话，最后获得总结反馈
- 高频动作：选择场景、发起练习、查看反馈
- 关键痛点：缺少可重复练习环境，难以对比前后进步

## 3. 核心能力

| 能力 | 为什么必须有 | MVP 是否包含 | 备注 |
|------|--------------|--------------|------|
| 角色与场景配置 | 决定练习内容 | 是 | 先支持预置模板 |
| 多轮对话练习 | 核心产品体验 | 是 | 支持基础上下文保留 |
| 练习后反馈 | 帮助用户复盘 | 是 | 先提供结构化文本反馈 |
| 历史记录与对比 | 提升长期价值 | 否 | 放到后续阶段 |

## 4. MVP 范围

- 本期必须做：预置角色模板、练习会话、多轮对话、结束反馈
- 可以后放：自定义角色、会话历史、分数系统、分享
- 明确不做：多人协作、复杂社交网络、移动端原生应用

## 5. 关键灰区决策

| 主题 | 当前选择 | 备选方案 | 采用 / 放弃原因 | 备注 |
|------|----------|----------|-----------------|------|
| 角色来源 | 先用预置模板 | 一开始支持自定义角色 | MVP 先降低建模复杂度 | 后续可扩展 |
| 反馈方式 | 先给结构化文本反馈 | 评分 + 图表 | 文本反馈更易快速落地 | |
| 练习结束条件 | 用户主动结束 | 系统自动限制轮数 | 保持体验简单 | |

## 6. 关键流程

### 6.1 主流程

1. 用户选择练习场景与角色模板
2. 系统初始化练习会话
3. 用户与角色进行多轮对话
4. 用户结束练习
5. 系统生成结构化反馈

### 6.2 边界与异常

- 异常 1：模型响应异常时需提示重试
- 边界 1：反馈只针对当前会话，不做长期能力判断

## 7. 项目约束

- 时间/资源约束：优先做单人 Web MVP
- 兼容性约束：首期只支持 Web
- 环境约束：依赖 LLM 对话能力
- 已知技术禁区：首期不做复杂实时协作
